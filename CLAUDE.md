## Architecture

Go TUI for managing git worktrees across repositories. Uses **Bubble Tea** (MVC pattern) with **lipgloss** for styling.

**Data flow:** `main` reads `WORKTREE_ROOT` env var → `scanner.Scan()` discovers repos → `model.New(repos)` creates Bubble Tea model → `Init()` fires async `fetchWorktrees` (default mode) → `ui.Render()` draws two-pane layout.

- **`cmd/wtui/main.go`** — Entry point. Wires scanner output into the Bubble Tea program (alt-screen mode).
- **`actions/`** — Executes git operations. `RemoveWorktree`/`ForceRemoveWorktree` run `git worktree remove` (with/without `--force`). `PruneWorktree` runs `git worktree prune`. `DeleteBranch`/`ForceDeleteBranch` run `git branch -d`/`-D`. `DropStash` runs `git stash drop stash@{N}`. `CopyToClipboard(text)` runs `pbcopy`. `OpenTerminal(path)` runs `open -a Terminal <path>`. `OpenVSCode(path)` runs `code <path>`.
- **`scanner/`** — Discovers git repos under `WORKTREE_ROOT` (default `~/dev`), up to 2 levels deep. Excludes `*-worktrees` dirs. Detects `.git` directories only (`.git` files are worktree markers and are excluded so worktree checkouts don't appear as separate repos). Returns repos sorted case-insensitively.
- **`gitquery/`** — Queries git data. `ListWorktrees(repoPath)` parses `git worktree list --porcelain` and enriches each worktree with stale detection and dirty status; returns main worktree first. `ListBranches(repoPath)` uses `git for-each-ref` to discover all local branches, then per branch queries upstream status, ahead/behind counts, unpushed commits, and worktree dirty status (files changed, lines added/deleted). Branches can carry multiple worktree paths. `FlattenBranches(branches)` expands multi-worktree branches into one `BranchRow` per path. `BranchDiff(worktreePath)` returns `git diff HEAD` for a worktree. `ListStashes(repoPath)` runs `git stash list --format=%gd%x00%ai%x00%s`. `StashDiff(repoPath, index)` runs `git stash show -p stash@{N}`. `ListCommits(repoPath)` returns up to 50 recent commits. `CommitDiff(repoPath, hash)` returns `git show <hash>`. `ListReflog(repoPath)` runs `git reflog --format=%h%x00%gd%x00%ar%x00%gs -n 50` and returns up to 50 `ReflogEntry{Hash, Selector, Date, Subject}`. `ReflogDiff(repoPath, hash)` runs `git diff <hash>^ <hash>`, falling back to `git show <hash>` for root commits. Only list-level failures are hard errors; per-item failures silently default to zero values.
- **`model/`** — Bubble Tea Model with two-pane focus system. `activePane` (0=left repos, 1=right content) determines which pane receives key input. `tab` toggles focus between panes. Five modes: `ModeWorktrees` (1), `ModeBranches` (2), `ModeStashes` (3), `ModeHistory` (4), `ModeReflog` (5). Default mode is `ModeWorktrees`. **Destructive mode** (`destructive` bool, default false): `Shift+D` toggles; when off, `d` key is a no-op (read-only safety gate). `Shift+D` is blocked while any overlay is active. **Left pane** (`activePane==0`): `up/down` (`j/k`) navigate repos (resets right-pane cursors, fires fetch); `repoScroll` tracks viewport offset via `ensureRepoVisible()` using `height - ui.RepoContentOverhead`; all other keys are no-ops except `tab`, `Shift+D`, and `q/esc`. **Right pane** (`activePane==1`): full key handling — `up/down` navigate items in the current mode, `left/right` (`h/l`) and `1/2/3/4/5` switch modes, `enter` opens diffs (dirty+worktree branches, stashes, commits, or reflog entries), `d` deletes with confirmation (requires destructive mode), `p` prunes stale worktrees (branches mode, requires destructive mode), `t`/`c` open terminal/VSCode at worktree path, `y` copies commit hash (history/reflog mode). Each mode has its own cursor (`worktreeSelected`, `branchSelected`, `stashSelected`, `commitSelected`, `reflogSelected`) and scroll (`worktreeScroll`, `branchScroll`, `stashScroll`, `commitScroll`, `reflogScroll`) tracked via `ensureWorktreeVisible()`, `ensureBranchVisible()`, `ensureStashVisible()`, `ensureCommitVisible()`, `ensureReflogVisible()`. `worktreeScroll` uses `height - ui.WorktreeContentOverhead`. `branchScroll` uses `height - ui.BranchContentOverhead` (status bar + 2 borders + mode header with separator). `stashScroll` uses `ui.StashLineCount()` to account for long messages that wrap to two lines. On delete failure, `DeleteFailedMsg` triggers a red-text force confirm dialog. `d` in stash mode → `DropStash` confirm. **Branch filtering:** `handleBranchResult` filters out worktree branches (`IsWorktree == true`) except the root branch (where `WorktreePath == repoPath`), which is pinned to position 0. The root branch cannot be deleted (`confirmBranchDelete` returns early). Result messages update state with stale-result protection via `isCurrentRepo()`. Confirm overlay intercepts all keys: `y`/`enter` executes, `n`/`q`/`esc` cancels. Code is split into focused handler methods: `handleConfirmKey`, `handleLeftPaneKey`, `handleOverlayKey`, `handleRightPaneKey`, plus action-specific methods (`handleEnter`, `handleDelete`, `handlePrune`, `confirmBranchDelete`, `confirmStashDrop`). Tests split across `model_test.go` (navigation), `model_action_test.go` (actions/confirm), `model_view_test.go` (view output).
- **`ui/`** — Stateless rendering. Two-pane layout with lipgloss borders: left pane (30 chars, full 4-sided border) + right pane (full 4-sided border). Active pane border = blue (color "12"), inactive = grey (color "238"). In destructive mode, right pane border = red (color "9") regardless of focus. `RenderParams` includes `ActivePane int`, `Destructive bool`, `RepoScroll int`, `BranchScroll int`, `StashScroll int`, `WorktreeScroll int`, `CommitScroll int`, `Worktrees []gitquery.Worktree`, `WorktreeSelected int`, `Commits []gitquery.Commit`, `CommitSelected int`, `Reflogs []gitquery.ReflogEntry`, `ReflogSelected int`, `ReflogScroll int`. Left pane scrolls via `RepoScroll`; long repo names are truncated with `truncateToWidth` to prevent line wrapping. Right pane has a mode header as its first content line (`renderModeHeader` shows `[1] worktrees  2 branches  3 stashes  4 history  5 reflog`). Below the header: mode 1 shows worktree details with `renderWorktreePane` (branch name as primary, path as secondary, blue `[root]` for main, dirty/clean/stale indicators), mode 2 shows branch details with `renderBranchPaneSelected` (branch status indicators, worktree labels, unpushed commits), mode 3 shows stash list with `renderStashPane` (date + message with wrapping), mode 4 shows commit history with `renderCommitPane` (hash, author, date, subject), mode 5 shows reflog with `renderReflogPane` (hash, selector, date, subject). Stash entries with long messages always wrap to two lines; `StashLineCount(msg, paneWidth)` returns 1 or 2 based on message width vs `StashPrefixWidth` (15). `splitAtWidth` splits messages at the visible-width boundary. Branch pane highlighting uses absolute branch index (all branches selectable). The branch pane shows the root branch (pinned to position 0 with blue `[root]` annotation via `rootStyle`) and non-worktree branches; worktree branches are filtered out by the model. Full-screen diff overlay replaces two-pane when active. Reflog diff overlay (`OverlayReflogDiff`) shows "No changes at this reflog entry" when the diff is empty (e.g. checkout entries). Confirmation dialog overlay (`Overlay==3`) shows a centered prompt; `ConfirmForce=true` renders it in red for force-delete dialogs. `RenderStatusBar(width, mode, overlay, activePane int, destructive, staleSelected bool)` renders hints only (no mode tabs — those moved to the right pane header). `tab: pane` and `q/esc: quit` appear first in all modes. In read-only mode (destructive=false), `D: destructive mode` hint appears; delete/drop hints are hidden. In destructive mode, `d: delete`/`d: drop` hints render in red. Mode 2 (branches) shows indicator legend; action hints (`t: terminal`, `c: code`) only appear when right pane is active. Branch status indicators: green `✔` (clean/even), yellow `●` (ahead/behind with `+N/-N` counts), red `●` (dirty with `N files +X/-Y`), purple `●` (no upstream). Indicators stack. Up to 5 unpushed commits shown with overflow. Worktree pane indicators: green `✔` (clean), red `●` (dirty with `N files +X/-Y`), red `✗ stale` (missing directory). Main worktree shows blue `[root]` annotation. Constants: `LeftPaneWidth = 30`, `RepoContentOverhead = 3`, `WorktreeContentOverhead = 5`, `BranchContentOverhead = 5`, `StashPrefixWidth = 15`.

## CI

CI runs on push to `main` and PRs targeting `main`. Checks: `gofmt`, `make test`, `make build`.

## Testing

Tests use real temp directories with actual `.git` dirs/files — no mocks. Scanner tests create nested repo structures; model tests simulate key messages via Bubble Tea's `Update()`. Gitquery tests create real git repos with remotes, commits, worktrees, and stashes to verify dirty/clean, ahead/behind, stale detection, and unpushed detection.
